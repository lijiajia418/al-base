## 描述

speech-realtime 页面 3 厂商并排对比时，Azure 因无中间结果导致录音期间该列长时间空白，与讯飞/腾讯的实时逐词反馈形成体验割裂。经调研确认 Azure 语音评估 API 设计如此（"流式"仅指音频输入，结果只在识别完成后一次性返回）。需优化前端展示以正确传达这一差异。

## 目标与验收

**目标**: 优化 Azure 列的等待态展示，让用户无需解释即可理解 3 家评测模式的差异

**业务价值**: 消除 3 厂商对比时的体验割裂，提升评测工具的可用性和专业感

**验收条件**:
- [ ] Azure 列录音期间显示等待态动画+说明文案，不再显示空白 "—"
- [ ] 三列 Provider 卡片有模式标签（实时评测/整句评测）
- [ ] 结果区域有响应时间轴对比
- [ ] 讯飞/腾讯现有流式行为不受影响

- **依赖项**: speech-realtime 现有实现（已就绪）

## 备注

Azure Speech SDK 的 `PronunciationAssessmentWithStream` 中 "stream" 指音频输入方式（PushStream），评估结果始终是 `recognizeOnceAsync` 一次性返回。这是 API 设计决策，非 bug。
