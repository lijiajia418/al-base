# F-002 用户体系

## 目标

为 A-Level 雅思课后 AI 教练系统搭建完整的用户体系基座，包括认证登录、角色管理、权限控制、组织管理（学校/班级/老师/学生/家长关系），支撑后续所有业务功能的用户身份和数据隔离需求。

## 目标用户

| 角色 | 使用场景 |
|------|---------|
| 学校管理员 | 管理老师、班级、配置学校设置、维护角色权限 |
| 老师 | 管理班级学生、添加家长、创建班级 |
| 学生 | 登录系统、查看自己的信息 |
| 家长 | 登录系统、查看绑定孩子的信息（只读） |

## In Scope（本次要做）

### 认证系统
- 手机号 + 短信验证码登录（自动注册）
- Opaque Token + Redis 会话管理（7天TTL、滑动续期）
- Auth 中间件链（限流 → 认证拦截器 → 角色解析 → 权限校验）
- 登出、角色切换

### 角色管理
- 4 种身份角色：school_admin / teacher / student / parent
- 一人多角色支持（如老师同时是家长）
- 角色生命周期管理（pending_activation → active → resigned/graduated/revoked）

### 权限系统
- 三层判定：身份角色（代码） → 职能权限（数据库） → 资源归属（关系表）
- teacher_roles + permissions + teacher_role_permissions 数据驱动权限
- 14 条系统预置权限（global 4 + class 10）
- 后台可维护角色和权限配置

### 组织管理
- 学校信息管理
- 班级 CRUD（管理员和老师都可创建）
- 老师管理（管理员添加/编辑/停用老师）
- 学生管理（老师手动添加到班级）
- 家长绑定（老师为学生添加家长）
- 租户隔离（行级隔离，school_id 条件）

### API 层
- 35 个 RESTful API，8 个模块
- 统一响应格式、错误码规范
- 分页、手机号脱敏

## Out of Scope（本次不做）

- 家庭模式（school_id=NULL、家长创建学生、上传学校进度） → v2
- 头像上传、换绑手机号 → v1.1
- 批量导入学生（Excel） → v1.1
- 邀请码机制 → 待需求确定
- 微信登录、邮箱登录 → 未来
- UI 界面实现 → 后续单独设计
- 业务功能（任务编排、训练、AI 教练等）

## 成功指标

| 指标 | 目标 |
|------|------|
| 验证码登录闭环 | 发码 → 登录 → Token → 访问受保护 API → 登出，全流程跑通 |
| 角色切换 | 多角色用户可切换身份，数据可见性随角色变化 |
| 权限校验 | teacher 不同职能角色在不同班级有不同操作权限 |
| 租户隔离 | 不同学校的数据完全隔离，跨校不可见 |
| API 覆盖 | 35 个 API 全部可用，错误场景有合理返回 |

## MVP 范围约束

| 约束 | 说明 |
|------|------|
| 手机号 = 唯一标识 | 所有用户必须有独立手机号，不允许共用 |
| 学校管理员 = 运营初始化 | 数据库直接初始化，不做自助注册 |
| MVP 聚焦学校模式 | 家庭模式推迟到 v2 |
| 家长端 = 只读看板 | 家长只查看，不做写操作 |
