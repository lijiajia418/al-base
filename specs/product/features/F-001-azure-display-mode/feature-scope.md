# Azure 发音评测展示模式优化

**Feature ID**: F-001-azure-display-mode
**状态**: draft
**创建时间**: 2026-04-02

---

## 1. 概述

**一句话描述**: 优化 speech-realtime 页面 Azure 列的展示方式，消除 3 厂商对比时因评测模式差异（流式 vs 整句）导致的体验割裂。

**背景**:
speech-realtime 页面并排展示 Azure、讯飞、腾讯三家语音评测结果。讯飞和腾讯支持流式中间结果（逐词实时返回分数），但 Azure 的 `recognizeOnceAsync` 只在识别完成后一次性返回结果。这导致录音期间 Azure 列长时间显示空白 "—"，用户容易误以为连接失败或服务异常。经调研确认这是 Azure API 的设计决策，非 bug。

**目标用户**:
- 使用语音评测对比工具的开发者/测试人员

---

## 2. 范围

### In Scope（本次要做的）

- Azure 列录音期间显示等待态 UI（脉冲动画 + 说明文案「录音结束后返回评测结果」）
- 录音结束后切换文案为「评测中...」
- 三列 Provider 卡片标题增加模式标签（讯飞/腾讯：「实时评测」蓝色；Azure：「整句评测」灰色）
- 结果区域增加响应时间轴对比（首词时间 + 最终结果时间）

### Out of Scope（本次不做的）

- 不伪造 Azure 中间结果（不在前端模拟逐词评测动画）
- 不改后端逻辑（streaming-session.ts、speech.handlers.ts 不动）
- 不改讯飞/腾讯的现有流式行为
- 不改变数据格式（SpeechIntermediatePayload 接口不变）

---

## 3. 边界定义

### 本 Feature 负责

- ProviderPanel 组件的等待态展示逻辑
- 模式标签的渲染
- 响应时间轴的计算和展示

### 本 Feature 不负责

- 各 Provider 的评测结果准确性
- 后端音频分发和结果收集逻辑
- WebSocket 连接管理

### 依赖的其他 Feature

| Feature | 依赖内容 | 说明 |
|---------|----------|------|
| speech-realtime 现有实现 | ProviderPanel 组件、SpeechIntermediatePayload 类型 | 已就绪 |

### 被依赖情况

无

## 4. 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|----------|
| Azure 列空白时间 | 0（录音期间始终有视觉反馈） | 人工验证 |
| 模式差异可识别性 | 用户无需解释即可理解 | 标签文案清晰 |
| 结果延迟可比性 | 3 家响应时间可量化对比 | 时间轴展示 |
